The current route calls `undo_last_command`, replays, formats, and returns `{status, message, data}`.

Replace the try block return statement with the extended version that:
1. Detects whether a command was actually soft-deleted by checking before and after: query count of is_active=True commands before and after `undo_last_command` to set `undone = (count_after < count_before)`. Alternatively, since `undo_last_command` silently no-ops when no commands exist, use a simpler approach: capture the command count from `session['commands']` after `get_session` — if commands list is empty, `undone=False`; else `undone=True`. This is accurate because `get_session` only returns `is_active=True` commands: if undo succeeded, the list shrinks; but `get_session` is called AFTER undo, so the returned commands already reflect the post-undo state. To know whether undo actually did something, check if the post-undo session has fewer commands than existed before — simpler: peek before calling undo.

Concrete implementation:

```python
@excel_bp.post('/undo')
@jwt_required()
def undo_excel():
    current_user_id = get_jwt_identity()
    session_id = request.form.get('session_id')

    if not session_id:
        return jsonify({"error": "Faltan datos (session_id)"}), 400

    try:
        # Peek at command count before undo to detect no-op
        pre_session = StateManager.get_session(session_id, current_user_id)
        had_commands = len(pre_session['commands']) > 0

        StateManager.undo_last_command(session_id)

        session = StateManager.get_session(session_id, current_user_id)
        current_df = _replay_session(session)
        data = ExcelService.format_dataframe_response(current_df)

        # Chart sync — re-execute active chart code post-undo (mirrors get_conversation_state)
        chart_data = None
        has_chart = False
        chart_code = StateManager.get_active_chart_code(session_id)
        if chart_code:
            try:
                chart_json = CodeExecutionService.execute_chart_generation(current_df, chart_code)
                chart_data = json.loads(chart_json)
                has_chart = True
            except Exception as chart_err:
                print(f"[UNDO] Chart re-execution failed: {chart_err}")

        return jsonify({
            "status": "success",
            "message": "Deshacer exitoso",
            "data": data,
            "chart_data": chart_data,
            "has_chart": has_chart,
            "undone": had_commands
        }), 200
    except ValueError as e:
        return jsonify({"error": str(e)}), 400
    except Exception as e:
        return jsonify({"error": str(e)}), 500
```

Note: `CodeExecutionService` is already imported at the top of excel.py (used in transform_excel). No new imports needed. `json` is also already imported.
