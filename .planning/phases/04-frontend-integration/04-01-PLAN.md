SheetJS 0.18.5 parse pattern (verified against installed package):
```typescript
import * as XLSX from 'xlsx';
// In browser, must use FileReader + readAsArrayBuffer — NOT XLSX.read(file, {type:'file'})
const reader = new FileReader();
reader.readAsArrayBuffer(file);
reader.onload = (e) => {
  const data = new Uint8Array(e.target!.result as ArrayBuffer);
  const workbook = XLSX.read(data, { type: 'array' });
  const ws = workbook.Sheets[workbook.SheetNames[0]];
  const rawRows = XLSX.utils.sheet_to_json<unknown[]>(ws, { header: 1, blankrows: false, defval: null });
  // rawRows[0] = header row, rawRows[1..] = data rows
};
// Merged cells: ws['!merges'] exists but react-data-grid beta does not support row-span.
// For v1: ignore merges — sheet_to_json already outputs flat values (covered cells appear null).
```

interface ExcelPreviewProps {

Logic:
1. Internal state: `parsedData: { columns: string[], rows: Record<string, unknown>[] } | null` and `isParsing: boolean`.
2. `useEffect` on `file` prop: if `file` is non-null and `gridData` is null/undefined, run the SheetJS parse (FileReader + readAsArrayBuffer pattern from research). On success set `parsedData`. On error call `console.error`.
